# 基本情绪框架
## 框架设计原则
‌‌‌‌　　在查阅了相关资料后（[ChatGPT深度研究：设计情绪系统综述](assets/个人系统/个人执行系统/情绪系统/情绪系统：综述/现代心理学中的情绪调节模型综述与应用分析.pdf)），情绪系统设计应该包含以下几个核心点：
- 底层稳态与唤醒调节（生理—神经层）
- 评估与情绪生成（评价层）
- 策略地图（过程层）
- 调度器（灵活性层）
- 概念与语言（构建与细粒度层）
- 技能化落地（临床模块层）
‌‌‌‌　　
‌‌‌‌　　经过我的多方面考虑，我认为应该如下理解其系统设计原则：
- 生理-神经层：
	- 使用其他的系统来维持基本的运转，但是利用本系统来评估不同的生理状态对情绪的影响，以及应对的策略。
		比如：**晨起-睡觉计划、运动计划、Daily Time Block等**。
	- 值得一提的是，很多生物神经原理其实蕴含在这里，可以利用其进行系统设计。比如：
		- 杏仁核是情绪中心、可以被训练。
		- 认知控制发生在PFC。但是在产生严重杏仁核反应的时候，PFC会失效80%。
- 评价层
	- 建立可靠的评价系统，然后利用这个系统来作用自己的情绪。
		- 现在的评价体系是：**“个价体”**。
- 过程层
	- 也是本系统设计的核心重点：如何通过系统设计，来提供干预点分类。
- 灵活性层
	- 强调策略，并且根据不同的情景来对其进行更新，也就是说系统必须要自己完善自己。
- 构建与细粒度层：
	- 情绪是人类最精细的东西，融合了多维度的认知、环境、生物神经调控、人类文化、个人认同……这些东西，绝对不能用一个简单的系统去建模。
	- 但是如果系统设计的过于复杂、会导致系统失效、从而失去其存在的意义：让我可以很好地控制情绪。所以，系统必须设计为可迭代，而且足够具有包容性。
		- 必要时，可以借助多模态的AI模型。

## 框架内容
‌‌‌‌　　基本情绪框架，是基于现在的系统设计下，对于<u>情绪处理认知</u>的结构体现。这个系统包含了：情绪调节的反馈回路，其中核心观点是，“利用理性脑作为调节器，去调节情绪反应、从而达到理想的状态。”
‌‌‌‌　　其核心的组成成分，为：
- **情绪认知系统**。主要包括：
	- **结构性压力控制**
	- **结构性情绪控制**
- **情绪监督-反馈系统**
	- 基于ChatGPT和Notion AI的：情感系统数据库

## ‌‌‌‌结构化情绪/压力控制——设计原则
‌‌‌‌　　作为情绪系统的核心，我认为结构化情绪系统建模一定要反复斟酌。并且，一定要遵循以下的设计策略：
- 系统必须具有可调控性——分级调节
	- 如情绪太浓烈、严重影响认知系统，先主动放松。如果情绪没有那么浓烈，认知系统可以介入。
- 认知可升级，且有些认知必须和理论库相关联。
	- 认知在这里我觉得应该是叫：理论加工之后的成果。不能满口理论而不去应用，也不能闭门造车。
- 必须同时处理：情绪Emotion和行动Action。
	- 比如遇到了一个非常令我生气的事情，必须冷静下来、解决情绪上的问题，而且一定要行动起来、去解决事情问题本身。
- 必须留存情绪缓冲。
